Coding Period — Week 2 (08 June – 14 June)

Week 2 focused on user authentication so only authorized operators can upload or run tours on the cluster.

Delivered:

  • User model (name, email, password hash, role, createdAt).
  • Register / login / logout routes with JWT issuance.
  • bcrypt password hashing and Express auth middleware.
  • Role hooks (admin vs operator) for future RBAC.

Discussed with Marc how long-lived tokens should behave during long tour executions. Next: MongoDB tour metadata models.
